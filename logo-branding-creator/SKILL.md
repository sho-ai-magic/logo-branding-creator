# logo-branding-creator

起業家・個人事業主などブランディング初心者が、ロゴとブランドアイデンティティをゼロから作り上げるための段階的ワークフロースキルです。

## 重要

- CRITICAL: 最初に必ずヒアリング（Phase 0）を実施すること。ヒアリングなしでロゴ案を出してはいけない
- CRITICAL: Phase 0の質問は必ず1問ずつ出す。複数の質問を同時に投げかけてはいけない
- CRITICAL: 各フェーズは必ずユーザーの確認・承認を得てから次に進む（一気に進めない）
- CRITICAL: 選択肢付きの質問では、必ず番号付きリストで選択肢を提示し、番号または「その他」で回答できるようにする
- CRITICAL: ユーザーが「その他」を選んだ場合は、内容を自由入力で求める
- 専門用語はすべて初心者にわかりやすく説明を添える
- ユーザーが迷っている場合は具体的な選択肢と推薦を提示する
- 各フェーズで「次に進みますか？」と確認を取ること

---

## ワークフロー概要

```
Phase 0: ヒアリング（共通情報を一括収集）
    ↓
Phase 1: ブランドアイデンティティ確立
    ↓
Phase 2: ロゴコンセプト複数案生成（AIプロンプト付き）
    ↓
Phase 3: ロゴ批評・絞り込み
    ↓
Phase 4: ブランドガイドライン生成
    ↓
Phase 5: 最終ロゴ決定
```

---

## Phase 0: ヒアリング

**目的**: 後続の全フェーズで使い回す共通情報を、1問ずつステップバイステップで収集する。

### ルール
- CRITICAL: 質問は必ず1問ずつ出す。次の質問は前の回答を受け取ってから出す
- 選択肢がある質問は番号付きリストで提示し、「番号で回答してください」と伝える
- 選択肢の最後には必ず「9. その他（自由に入力）」または「0. おまかせ（Claudeに提案してもらう）」を加える
- 複数選択可の場合は「複数選択OK（例：1, 3, 5）」と明示する
- 「その他」が選ばれたら「どんな〇〇ですか？自由に教えてください」と内容を聞く

### 質問の順序と形式

**Q1 — ブランド名**（自由入力）
```
ブランディングを始めましょう！
まず最初の質問です。

ブランド名（またはビジネス名）を教えてください。
まだ決まっていない場合は「未定」と入力してください。一緒に考えることもできます。
```
→ 「未定」の場合は「業界とターゲットを教えてもらえればブランド名の候補を出せます。まずは次の質問に進みましょう！」と伝えて続行。

---

**Q2 — 業界・ジャンル**（選択肢＋自由入力）
```
Q2 / 7：業界・ジャンルはどれに近いですか？
番号で教えてください。

1. 飲食・カフェ・レストラン
2. 美容・ヘアサロン・エステ・ネイル
3. ファッション・アパレル・ハンドメイド
4. コーチング・コンサルティング・教育
5. テクノロジー・ITサービス・アプリ
6. 健康・フィットネス・ウェルネス
7. クリエイティブ・デザイン・写真・映像
8. 小売・EC・雑貨
9. その他（自由に入力してください）
```

**Q3 — ターゲット客層**（自由入力）
```
Q3 / 7：ターゲットのお客様はどんな方ですか？
自由に教えてください。

例：

・中小企業の経営者・マネージャー
・子育て中のママ
・健康意識の高い40代男性

わからなければ「おまかせ」でも大丈夫です。
```

```
例：

・オンラインビジネスコーチング
・オリジナルアクセサリーのハンドメイド販売
・中小企業向けのWebマーケティング支援
```

**Q5 — ブランドの印象・雰囲気**（番号選択・複数可）
```
Q5 / 7：このブランドにどんな印象を持たせたいですか？
番号で教えてください（複数選択OK、例：1, 3, 7）

1. プレミアム・ラグジュアリー（高級感、特別感）
2. フレンドリー・親しみやすい（温かみ、身近な存在）
3. クール・スタイリッシュ（洗練された、かっこいい）
4. 遊び心・ポップ（楽しい、カラフル、個性的）
5. ナチュラル・オーガニック（自然、エコ、癒し）
6. プロフェッショナル・信頼感（誠実、実績、安心）
7. ミニマル・シンプル（シンプル、無駄のない、洗練）
8. 大胆・インパクト重視（目を引く、強い、記憶に残る）
9. 未来的・テクノロジー感（革新的、先進的、デジタル）
10. その他（自由に入力してください）
```

**Q6 — カラーの好み**（選択肢＋自由入力）
```
Q6 / 7：色のイメージはありますか？

1. 暖色系（赤・オレンジ・黄色など）
2. 寒色系（青・水色・紺など）
3. アースカラー系（茶・ベージュ・カーキなど）
4. モノクロ（黒・白・グレー）
5. パステル系（淡いピンク・ミント・ラベンダーなど）
6. ビビッド・鮮やか系
7. 特定の色がある（→番号の後に色名も教えてください）
8. 避けたい色がある（→番号の後に色名も教えてください）
9. おまかせ（Claudeに提案してもらう）
```
→ 「7」または「8」が選ばれたら「どんな色ですか？」と追加で聞く。

---

**Q7 — ロゴのスタイル**（番号選択）
```
Q7 / 7：最後の質問です！
好みのロゴスタイルはありますか？

1. ワードマーク — ブランド名を文字だけでデザイン（例：Google、Sony）
2. モノグラム — イニシャル文字を組み合わせたデザイン（例：LV、Chanel）
3. アイコン＋テキスト — シンボルマークと文字を組み合わせ（例：Apple、Nike）
4. エンブレム — バッジやシールのような形（例：Starbucks、Harvard）
5. アブストラクト — 抽象的・幾何学的なシンボル（例：Pepsi、Adidas）
6. おまかせ（Claudeに最適なスタイルを提案してもらう）
7. その他（自由に入力してください）
```

## Phase 1: ブランドアイデンティティ確立

**目的**: ブランドの骨格を作る。ロゴの方向性を決める前提となる情報を整理する。

ヒアリング情報をもとに以下を生成する：

### 出力項目

**1. ブランドパーソナリティ（5つの特性）**
- 例：「温かい・誠実・革新的・洗練された・遊び心がある」
- 各特性に一言説明を添える

**2. トーン・オブ・ボイス（伝え方のトーン）**
- SNS・ウェブサイトでどんな言葉づかいをするか
- 例：「丁寧だけど堅すぎない、友人のようなアドバイザーとして話す」

**3. ミッションステートメント（1〜2文）**
- 「誰のために、何を、なぜ提供するか」
- 例：「私たちは〇〇に悩む△△のために、□□を通じて◇◇を実現します」

**4. ポジショニングステートメント**
- 競合との差別化を一言で
- 例：「〇〇業界において、△△という点で唯一□□を提供するブランド」

**5. タグライン候補（3案）**
- キャッチコピー的なもの
- 短く、記憶に残るもの

**6. ブランドワード（10語）**
- このブランドに連想してほしい言葉
- 例：「温かさ、信頼、丁寧、こだわり、本物、...」

**7. ロゴが避けるべきスタイル**
- このブランドに合わないビジュアルの方向性

**8. ロゴが伝えるべきメッセージ**
- ロゴを見た人に感じてほしいこと

完了後：「ブランドアイデンティティができました！次はロゴコンセプトを複数案作成します。進めますか？」

---

## Phase 2: ロゴコンセプト複数案生成

**目的**: 異なる方向性の8つのロゴコンセプトを生成する（ユーザーが比較・選択できるように）。

### 出力形式

各コンセプトについて以下の情報を含める：

```
【コンセプト No.X】「コンセプト名」

◆ スタイル
（例：ミニマルワードマーク / モノグラム＋エンブレム / アイコン＋テキスト など）

◆ シンボル・文字のアイデア
（何をどう表現するか、具体的に）

◆ タイポグラフィ
（フォントのスタイル：セリフ/サンセリフ/スクリプト/ディスプレイ など）

◆ カラーパレット
（メインカラー・サブカラー・アクセントカラーのイメージ）

◆ 伝わる印象
（このロゴを見た人がどう感じるか）

◆ 最適な使用シーン
（どのメディア・用途に向いているか）

◆ AIロゴ生成プロンプト（日本語）
（Midjourney・DALL-E・Adobe Fireflyなどで使えるプロンプト）
```

### 8つのコンセプトの方向性（必ず異なるアプローチを取ること）

1. **シンプル・クリーン系**（汎用性重視）
2. **ラグジュアリー・プレミアム系**
3. **フレンドリー・親しみやすい系**
4. **モノグラム・イニシャル活用系**
5. **アイコン＋テキスト組み合わせ系**
6. **ミニマル・モダン系**
7. **ブランドの業界特性を活かした系**
8. **ユニーク・独自路線系**

完了後：「8つのコンセプトが揃いました。気になるコンセプトはありますか？次のステップで各コンセプトを批評し、絞り込みます。」

---

## Phase 3: ロゴ批評・絞り込み

**目的**: 生成したコンセプトを客観的に評価し、最終候補を2〜3案に絞る。

### ステップ3-1: ユーザーに「気になる案」を聞く

「8つのコンセプトの中で、気になるものや直感的に好きなものはありましたか？番号で教えてください（複数OK）」

### ステップ3-2: 全コンセプトを以下の基準でスコアリング（各5点満点）

| 評価軸 | 内容 |
|--------|------|
| 記憶に残りやすさ | 一度見たら覚えられるか |
| 独自性 | 他のブランドと差別化できているか |
| ターゲット適合性 | ターゲットオーディエンスに響くか |
| 汎用性 | 白黒・小サイズ・SNSアイコンでも使えるか |
| ブランド適合性 | Phase 1で定義したパーソナリティと一致しているか |
| シンプルさ | 過剰に複雑でないか |

### ステップ3-3: 上位2〜3案を選び、改善提案を加える

各上位案に対して：
- **強み**（このコンセプトの良いところ）
- **弱み**（改善すべき点）
- **改善提案**（具体的にどう修正すれば強くなるか）

### ステップ3-4: 推薦案を1つ明示する

「以上を踏まえ、私（Claude）が最もおすすめするのはコンセプトNo.Xです。理由：〜〜」

完了後：「絞り込みができました。次はブランドガイドラインを作成します。進めますか？」

## Phase 4: ブランドガイドライン HTML 生成

**目的**: Phase 0〜3で固めた全情報をもとに、Claudeが直接ブランドガイドラインを**単一の完全なHTMLファイル**として出力する。

### CRITICAL: このフェーズでは必ず完全な単一HTMLを出力すること。テキストでのガイドライン出力・React経由・API経由は不可。

### CRITICAL: 出力方法は実行環境に応じて自動で選ぶこと（両環境対応）

- **ファイル書き込みツールが使える環境（Claude Code など）の場合**：
  ブランド名をもとにした分かりやすいファイル名（例：`HARU-COFFEE-brand-guideline.html`）で、HTMLファイルをそのまま作業フォルダに保存する。保存後、ファイルのパスをユーザーに伝える。
- **アーティファクトが使える環境（Claude のチャット版など）の場合**：
  `.html` アーティファクトとして直接出力する。
- どちらの場合も、生成するHTMLの中身（構造・デザイン仕様）は完全に同一とする。

### 参照ファイル
- `assets/brand-guideline-sample.html` — 完成品のサンプルHTML。デザイン・構造・CSS変数設計・SVGロゴ実装・インタラクションの参考として必ず確認してから生成すること。
- `references/html-structure-guide.md` — セクション構成・CSS命名規則・SVGロゴパターンの早見表。

---

### 出力するHTMLの構成（必須セクション）

以下の6セクションを含む完全な単一HTMLファイルを生成する：

```
01 / LOGO SYSTEM    ロゴシステム
  - ライト背景用・ダーク背景用の2バリエーション表示
  - サブマーク（ファビコン・SNSアイコン用）複数サイズ展示（64/48/32/20/16px）
  - 使ってよいパターン / 使ってはいけない例（2カラムグリッド）

02 / COLOR          カラーパレット
  - カラースウォッチグリッド（各色にHEX・役割・使用比率）
  - 各色の用途説明テーブル

03 / TYPOGRAPHY     タイポグラフィ階層
  - Hero/H1/H2/H3/Body/Small/Caption の7段階タイプスケール
  - ディスプレイ用・本文用フォントペア紹介（Google Fontsリンク付き）

04 / BRAND          ブランドパーソナリティ
  - パーソナリティ特性カード（5枚グリッド）
  - タグライン3案（推薦案をハイライト・バッジ表示）
  - トーン・オブ・ボイス（心がけること/避けること 2カラム）
  - ミッションステートメント

05 / PREVIEW        コンテキスト別プレビュー
  - ライト背景プレビュー（見出し・本文・ボタン）
  - ダーク背景プレビュー（見出し・本文・ボタン）

06 / CHECKLIST      ブランドチェックリスト
  - onclick で checked クラスをトグルするインタラクティブ10項目
```

---

### HTMLのデザイン仕様（必ず遵守）

**構造**
- `<!DOCTYPE html>` から `</html>` までの完全な単一ファイル
- sticky トップバー（ロゴ＋セクションナビ `#logo #color #type #brand #preview`）
- ダーク Hero セクション（ブランドロゴ・タイトル・推薦タグライン）
- `max-width: 880px` のページコンテナ、`padding: 0 40px 120px`
- セクション間に `height: 0.5px` の divider
- `@media (max-width: 640px)` でグリッドを1カラムに

**CSSカラー変数（`:root` に定義）**
```css
:root {
  --primary:      /* Phase 3で決定したメインカラー HEX */
  --primary-mid:  /* メインカラーの中間色（明度+30%程度） */
  --primary-light:/* メインカラーの淡色（明度+60%程度） */
  --primary-pale: /* メインカラーの極淡色（背景用） */
  --dark:         /* ほぼ黒のテキスト色 */
  --dark-mid:     /* サブテキスト用ミッドグレー */
  --dark-light:   /* キャプション用ライトグレー */
  --off-white:    /* ページ背景・サーフェス色 */
  --white:        #FFFFFF;
  --border:       rgba(44,44,42,0.12);
  --border-dark:  rgba(44,44,42,0.24);
}
```

**Google Fonts の読み込み**
- Phase 3で決定したフォント2種を `<link>` で読み込む
- `--font-display` と `--font-body` の2変数で管理

**ロゴのSVG実装**（Phase 2〜3で決定したスタイルに応じて選択）
- ワードマーク型：`<text>` 要素でブランド名を描画
- アイコン＋テキスト型：幾何学シンボル（`<rect>/<circle>/<path>`）＋ wordmark
- モノグラム型：角丸矩形の背景＋イニシャル `<text>`
- 抽象型：ブランドの特性を表す幾何学図形の組み合わせ
- ライト背景用・ダーク背景用・サブマーク（アイコンのみ）の3バリエーションを作成

**チェックリストのインタラクション**
```html
<div class="checklist-item" onclick="this.classList.toggle('checked')">
  <div class="checklist-box"></div>チェック項目テキスト
</div>
```

### Claudeの出力手順

1. Phase 0〜3で確定したすべての情報（ブランド名・カラー・フォント・パーソナリティ・タグライン・ミッション・トーン・ロゴコンセプト）を整理する
2. 上記仕様に従ってHTMLを一から書く（テンプレートのコピーではなく、ブランド固有の内容で生成）
3. 実行環境に応じて出力する（上記「出力方法は実行環境に応じて自動で選ぶこと」を参照）：
   - ファイル書き込み可能な環境 → HTMLファイルとして作業フォルダに保存
   - アーティファクト対応環境 → `.html` アーティファクトとして出力
4. 生成完了後、ユーザーに以下を伝える（環境に合わせて文面を選ぶ）：

**ファイルとして保存した場合：**
```
ブランドガイドラインHTMLができました！
保存先：（ファイルのパス）
このファイルをブラウザで開けば、そのまま使えます。

次は最終ロゴの決定に進みましょう。
```

**アーティファクトとして出力した場合：**
```
ブランドガイドラインHTMLができました！
右上の「↓」ボタンからHTMLファイルをダウンロードできます。
ブラウザで開けばそのまま使えます。

次は最終ロゴの決定に進みましょう。
```

## Phase 5: 最終ロゴ決定

**目的**: すべての情報をもとに最終案を決定し、次のアクションを提示する。

### ステップ5-1: 最終候補の比較表を作成

| 項目 | 案A | 案B | 案C（あれば） |

## エラーハンドリング

**ユーザーが途中で迷った場合**
- 「どちらにしましょうか？」ではなく「私のおすすめは〇〇です。理由は〜」と積極的に推薦する

**ユーザーが「全部好き」「全部わからない」という場合**
- Phase 1のブランドアイデンティティに最も合致するコンセプトを基準に推薦する

**ユーザーが途中でフェーズをスキップしたい場合**
- 「Phase X を飛ばして進めることもできますが、〇〇の情報が後のフェーズで必要になります。スキップしますか？」と確認する

**ユーザーがブランド名を決めていない場合**
- 「まずブランド名を一緒に考えましょうか？業界とターゲットを教えてもらえれば、候補を出せます」と提案する

## 例

### 例1: カフェオーナーの場合

ユーザー：「カフェのロゴを作りたい。ブランド名は『HARU COFFEE』、ターゲットは近所の30代女性、ナチュラルで温かい雰囲気にしたい」

アクション：
1. Phase 0のヒアリングで不足情報（好きな色・スタイル）を確認
2. Phase 1でブランドアイデンティティ生成（「温かさ・手作り感・日常の豊かさ」など）
3. Phase 2で8案生成（コーヒーカップのシンボル案・葉っぱモチーフ案・文字だけ案など）
4. Phase 3で批評→上位3案に絞る
5. Phase 4でブランドガイドライン（アースカラー系・手書き風フォントなど）
6. Phase 5で最終決定→Canvaで作成するよう案内

### 例2: コーチング起業家の場合

ユーザー：「個人コーチングのブランディングをゼロから作りたい」

アクション：
1. Phase 0でブランド名・ターゲット・サービス内容・印象を収集
2. 以降フェーズ順に進める
